# 🚀 راهنمای استقرار 3X-UI Bot روی Cloudflare Dashboard

راهنمای کامل نصب ربات مدیریت 3x-ui روی Cloudflare Worker — بدون نیاز به Wrangler CLI.

---

## قدم ۱: ساخت KV Namespaces

به **دو** KV Namespace نیاز دارید:

1. وارد **[dash.cloudflare.com](https://dash.cloudflare.com)** شوید
2. از منوی سمت چپ، **Workers & Pages** را انتخاب کنید
3. روی تب **KV** کلیک کنید
4. دکمه **Create a namespace** را بزنید
5. نام `BOT_KV` را وارد کنید و **Create** را بزنید
6. دوباره **Create a namespace** را بزنید
7. نام `BOT_STATE` را وارد کنید و **Create** را بزنید

---

## قدم ۲: ساخت Worker

1. از منوی سمت چپ، **Workers & Pages** → **Overview**
2. دکمه **Create** را بزنید
3. گزینه **Create Worker** را انتخاب کنید
4. نام Worker را `3xui-bot` بگذارید
5. روی **Deploy** کلیک کنید (فعلاً با کد پیش‌فرض)
6. بعد از ساخت، روی **Edit Code** کلیک کنید

---

## قدم ۳: قرار دادن کد Worker

1. در ویرایشگر آنلاین، **کل کد موجود را پاک کنید**
2. محتوای فایل `3xui-bot-worker.js` را **کپی و پیست** کنید
3. دکمه **Deploy** را بزنید

---

## قدم ۴: اتصال KV Namespaces

1. به صفحه تنظیمات Worker بروید: **Workers & Pages** → `3xui-bot`
2. تب **Settings** را انتخاب کنید
3. بخش **Bindings** را پیدا کنید
4. روی **Add** کلیک کنید
5. نوع **KV Namespace** را انتخاب کنید
6. تنظیمات اول:
   - **Variable name**: `BOT_KV`
   - **KV namespace**: `BOT_KV` را انتخاب کنید
7. **Save** را بزنید
8. دوباره **Add** کلیک کنید
9. تنظیمات دوم:
   - **Variable name**: `BOT_STATE`
   - **KV namespace**: `BOT_STATE` را انتخاب کنید
10. **Save** را بزنید

---

## قدم ۵: تنظیم متغیرهای محیطی

در همان صفحه **Settings** → **Environment Variables**:

### ۵-الف: BOT_TOKEN (الزامی)

| تنظیم | مقدار |
|-------|-------|
| Variable name | `BOT_TOKEN` |
| Type | Encrypt (Secret) |
| Value | توکن ربات از @BotFather |

### ۵-ب: SUPPORT_USERNAME (اختیاری)

| تنظیم | مقدار |
|-------|-------|
| Variable name | `SUPPORT_USERNAME` |
| Type | Text |
| Value | آیدی تلگرام پشتیبانی (بدون @) |

⚠️ بعد از تنظیم این متغیر، دکمه **🎧 پشتیبانی** در منوی ربات ظاهر میشه.

### ۵-ج: PANELS_JSON (اختیاری)

اگه می‌خواهید پنل‌ها رو از ابتدا تنظیم کنید (به جای استفاده از `/addpanel`):

| تنظیم | مقدار |
|-------|-------|
| Variable name | `PANELS_JSON` |
| Type | Encrypt (Secret) |
| Value | JSON زیر |

```json
{
  "botToken": "توکن_ربات",
  "adminChatIds": ["123456789"],
  "panels": [
    {
      "id": "server1",
      "name": "سرور اصلی",
      "panelUrl": "https://your-panel.com:54321",
      "apiToken": "توکن_API_پنل",
      "inboundIds": [1],
      "subBaseUrl": "https://sub.yourdomain.com",
      "subPath": "sub",
      "authType": "bearer"
    }
  ]
}
```

### ۵-د: متغیرهای تکی (اختیاری)

اگه نمی‌خواهید از `PANELS_JSON` استفاده کنید:

| Variable | نوع | مثال |
|----------|------|-------|
| `BOT_TOKEN` | Secret | `123456:ABC-DEF` |
| `PANEL_URL` | Text | `https://panel.example.com:54321` |
| `API_TOKEN` | Secret | `your-api-token` |
| `ADMIN_CHAT_IDS` | Text | `123456789,987654321` |
| `INBOUND_IDS` | Text | `1,2,3` |
| `SUB_BASE_URL` | Text | `https://sub.example.com` |
| `SUPPORT_USERNAME` | Text | `your_support` (بدون @) |

⚠️ بعد از هر تغییر متغیر، حتماً **Deploy** را بزنید!

---

## قدم ۶: تنظیم Cron Triggers

1. در صفحه تنظیمات Worker، تب **Settings**
2. بخش **Triggers** را پیدا کنید
3. روی **Add Cron Trigger** کلیک کنید
4. این ۵ Cron را اضافه کنید:

| Cron | توضیح |
|------|--------|
| `*/5 * * * *` | بررسی وضعیت Xray - هر ۵ دقیقه |
| `*/10 * * * *` | هشدار CPU/RAM - هر ۱۰ دقیقه |
| `0 9 * * *` | گزارش روزانه - هر روز ساعت ۹ |
| `0 */6 * * *` | بکاپ خودکار - هر ۶ ساعت |
| `*/30 * * * *` | بررسی درخواست‌های تمدید - هر ۳۰ دقیقه |

---

## قدم ۷: تنظیم Webhook تلگرام

بعد از Deploy، آدرس Worker چیزی شبیه این است:
```
https://3xui-bot.your-username.workers.dev
```

این آدرس را در مرورگر باز کنید:

```
https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook?url=https://3xui-bot.your-username.workers.dev/webhook
```

اگر پاسخ `{"ok":true}` بود، یعنی همه چیز درست است! ✅

---

## قدم ۸: تنظیم ادمین اولیه

**اولین کاری که باید بکنید** بعد از Deploy:

1. در تلگرام ربات را Start کنید
2. دستور `/makeadmin` را بفرستید
3. ربات شما را به عنوان **سوپر ادمین** ثبت می‌کند
4. حالا `/admin` را بزنید تا منوی مدیریت باز شود

اگر `adminChatIds` را در `PANELS_JSON` تنظیم کرده‌اید، خودکار سوپر ادمین شناخته می‌شوید.

---

## قدم ۹: افزودن پنل 3x-ui

اگر پنل را با `PANELS_JSON` تنظیم نکرده‌اید:

1. `/admin` را بزنید
2. دکمه **🖥️ مدیریت پنل‌ها** را بزنید
3. دکمه **➕ افزودن پنل جدید** را بزنید
4. مراحل زیر را طی کنید:
   - 📝 نام پنل (مثلاً: سرور آلمان)
   - 🔗 آدرس پنل (مثلاً: `https://de.example.com:54321`)
   - 🔑 توکن API پنل (از Settings → Security → API Token)
   - 🔗 آدرس اشتراک (SUB_BASE_URL) یا `0` برای رد کردن

---

## قدم ۱۰: افزودن ادمین پنل (سوپر ادمین)

برای دادن دسترسی محدود به دیگران:

1. `/admin` را بزنید
2. دکمه **👥 ادمین‌ها** را بزنید
3. دکمه **➕ افزودن ادمین پنل** را بزنید
4. مراحل تعاملی:
   - 💬 Chat ID ادمین جدید
   - 🖥️ انتخاب پنل‌ها (با کاما جدا کنید)
   - 📊 محدودیت تعداد کاربر (یا 0 برای نامحدود)

یا با دستور:
```
/addadmin <chatId> <panelId1,panelId2> [maxUsers]
مثال: /addadmin 123456789 US,DE 50
```

---

## قدم ۱۱: تست ربات

1. `/start` را بزنید
2. اگه سوپر ادمین هستید، منوی کامل باز میشه
3. اگه ادمین پنل هستید، منوی محدود باز میشه
4. اگه کاربر عادی هستید، ایمیل خود را وارد کنید
5. مصرف خود را ببینید! 🎉

## 🔧 سیستم نقش‌ها

### 👑 سوپر ادمین
- دسترسی کامل به همه پنل‌ها و قابلیت‌ها
- می‌تونه ادمین پنل اضافه/حذف کنه
- می‌تونه کاربران رو بن/تعلیق کنه
- هشدارها و بکاپ‌ها فقط به سوپر ادمین‌ها ارسال میشه
- قابل حذف نیست

### 🛠️ ادمین پنل
- فقط به پنل‌های اختصاصی داده شده دسترسی داره
- فقط کاربرانی که خودش ساخته رو میبینه
- محدودیت تعداد کاربر قابل تنظیم
- درخواست تمدید فقط از کاربران خودش بهش میرسه
- comment کاربر در پنل 3x-ui شامل `TG:<chatId>` ادمین هست

## 🔧 عیب‌یابی

### ادمین شناخته نمی‌شوم
- دستور `/makeadmin` را بفرستید (فقط وقتی هیچ ادمینی وجود ندارد)
- یا `adminChatIds` را در `PANELS_JSON` اضافه کنید
- Chat ID خود را از @userinfobot بگیرید

### ربات جواب نمی‌دهد
1. در داشبورد Cloudflare → Worker → **Logs** را چک کنید
2. آیا Webhook درست تنظیم شده؟
   ```
   https://api.telegram.org/bot<TOKEN>/getWebhookInfo
   ```
3. آیا KV Namespaces متصل شده؟ Settings → Bindings را چک کنید
4. آیا `BOT_TOKEN` تنظیم شده؟

### خطای "No panel configured"
- پنل را با `/addpanel` اضافه کنید
- یا `PANELS_JSON` را تنظیم کنید

### کاربر یافت نمی‌شود
- ایمیل باید دقیقاً همان چیزی باشد که در پنل 3x-ui ثبت شده
- `inboundIds` را چک کنید

### خطا در حذف کاربر
- در منوی ادمین → **📋 لاگ خطاها** را چک کنید
- همه خطاها در KV ذخیره میشن

### کاربران آنلاین ۰ نشون میده
- API باید `POST /panel/api/clients/onlines` رو پشتیبانی کنه
- نسخه 3x-ui رو آپدیت کنید

### ادمین پنل نمی‌تونه کاربری رو ببینه
- ادمین پنل فقط کاربرانی که خودش ساخته رو میبینه
- در پنل 3x-ui → comment کاربر باید `TG:<chatId>` داشته باشه

## 📱 پیدا کردن Chat ID

1. به ربات **@userinfobot** پیام بدهید
2. یا به **@RawDataBot** پیام بدهید
3. Chat ID عددی را در `adminChatIds` قرار دهید

---

## 🌐 استفاده از دامنه اختصاصی (اختیاری)

1. در Cloudflare → **Workers** → **Custom Domains**
2. دامنه خود را اضافه کنید (مثلاً `bot.yourdomain.com`)
3. Webhook جدید تنظیم کنید:
   ```
   https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://bot.yourdomain.com/webhook
   ```

## 📋 اطلاعات KV

| کلید | توضیح |
|------|--------|
| `user:<chatId>` | کاربر ثبت‌نام شده (شامل `createdBy`) |
| `userbackup:<chatId>` | بکاپ اطلاعات کاربر |
| `panels:config` | تنظیمات پنل‌ها |
| `admin:ids` | لیست ادمین‌ها |
| `admin:role:<chatId>` | نقش ادمین |
| `banned:<chatId>` | کاربر بن شده |
| `suspended:<chatId>` | کاربر تعلیق شده |
| `renewal:<id>` | درخواست تمدید |
| `error:<id>` | لاگ خطا |
| `alert:xray:<panelId>` | هشدار Xray |
| `backup:<panelId>` | زمان آخرین بکاپ |

## 🔄 آپدیت ربات

برای آپدیت به نسخه جدید:
1. کد جدید `3xui-bot-worker.js` را در ویرایشگر کپی کنید
2. **Deploy** را بزنید
3. نیازی به تنظیم مجدد متغیرها نیست

## 💡 نکات

- **هزینه**: Cloudflare Workers plan رایگان تا ۱۰۰,۰۰۰ درخواست در روز
- **KV**: رایگان تا ۱۰۰,۰۰۰ خواندن و ۱,۰۰۰ نوشتن در روز
- **پشتیبانی**: از منوی ادمین → **🎧 پشتیبانی** (اگه `SUPPORT_USERNAME` تنظیم شده)
- **دیباگ**: از منوی ادمین → **📋 لاگ خطاها** برای دیدن خطاهای اخیر
